# HW10-11 – компьютерное зрение в PyTorch: CNN, transfer learning, detection/segmentation

## 1. Кратко: что сделано

- **Часть A:** Выбран датасет **SVHN** (цифры 0-9 на фото домов). Причина: простой, быстрый (~400 МБ), 10 классов, есть в torchvision.
- **Часть B:** Выбран датасет **OxfordIIITPet**, трек **segmentation**. Причина: понятная задача (питомец vs фон), предобученная модель работает "из коробки", быстрее чем detection.
- **Сравнение в части A:** Простая CNN без/с аугментациями (C1/C2) vs ResNet18 с transfer learning (C3/C4).
- **Сравнение в части B:** Два режима постобработки масок (базовая vs удаление мелких компонент).

## 2. Среда и воспроизводимость

- **Python:** 3.12
- **torch / torchvision:** 2.7.1+cu118 / 0.22.1+cu118
- **Устройство:** GPU (CUDA)
- **Seed:** 42
- **Как запустить:** Открыть `HW10-11.ipynb` в Jupyter Lab и выполнить Run All.

## 3. Данные

### 3.1. Часть A: классификация

- **Датасет:** SVHN
- **Разделение:** train=58,606 (80%), val=14,651 (20%), test=26,032 (официальный split)
- **Базовые transforms:** ToTensor(), Normalize(SVHN_MEAN, SVHN_STD)
- **Augmentation transforms:** RandomHorizontalFlip(p=0.5), RandomCrop(32, padding=4)
- **Комментарий:** 10 классов (цифры), изображения 32×32, задача проще чем CIFAR-100/Flowers, но требует нормализации под SVHN.

### 3.2. Часть B: structured vision

- **Датасет:** OxfordIIITPet
- **Трек:** segmentation
- **Ground truth:** Маска с классами 0 (фон) и 1-3 (части питомца)
- **Предсказания:** Выход DeepLabV3_ResNet50, argmax по каналам
- **Комментарий:** Задача бинарной сегментации (питомец vs фон) разумна: модель предобучена на похожих данных, метрика IoU интерпретируема.

## 4. Часть A: модели и обучение (C1-C4)

- **C1 (simple-cnn-base):** SimpleCNN (3 Conv-блока), без аугментаций, обучалась 3 эпохи.
- **C2 (simple-cnn-aug):** Та же архитектура, с аугментациями (flip, crop), 3 эпохи.
- **C3 (resnet18-head-only):** ResNet18 с ImageNet-весами, заморожен backbone, обучалась только fc-голова, 3 эпохи.
- **C4 (resnet18-finetune):** ResNet18, разморожены layer4+fc, разные LR для backbone (1e-4) и головы (1e-3), 3 эпохи.

**Дополнительно:**
- **Loss:** CrossEntropyLoss
- **Optimizer:** Adam
- **Batch size:** 64
- **Epochs (макс):** 3 (FAST_MODE)
- **Критерий выбора лучшей модели:** Максимальная `best_val_accuracy`

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)

### Выбран segmentation track

- **Модель:** DeepLabV3_ResNet50 (pretrained на COCO)
- **Что считается foreground:** Все классы > 0 (питомец), класс 0 — фон
- **V1:** Базовая постобработка — прямой argmax выхода модели
- **V2:** Альтернативная — удаление компонент связности < 50 пикселей
- **Как считался mean IoU:** Binary IoU для foreground: `intersection / union` по маскам (pred > 0, gt > 0)
- **Дополнительные метрики:** Не считались (достаточно mean IoU по заданию)

## 7. Анализ

Простая CNN (C1) показала accuracy 0.364 на валидации — это ожидаемо для маленькой сети на 10-классовой задаче. Аугментации (C2) не дали улучшения, вероятно, из-за недостаточного числа эпох: модель не успела извлечь пользу из расширенных данных. Pretrained ResNet18 (C3) дала прирост до 0.298 даже при обучении только головы — это подтверждает силу transfer learning. Fine-tuning (C4) значительно улучшил результат до 0.642: разморозка layer4 позволила адаптировать признаки под специфику цифр в реальных условиях. Для сегментации выбрана метрика mean IoU, так как она стандартна для pixel-level задач и устойчива к дисбалансу классов. Переход от V1 к V2 не изменил IoU, что говорит об отсутствии мелких шумов в предсказаниях pretrained модели. Наиболее показательные ошибки: модель иногда сегментирует фон как питомца на сложных изображениях с перекрытиями.

## 8. Итоговый вывод

- **Базовый конфиг классификации:** ResNet18 с partial fine-tuning (layer4+fc) — даёт лучший баланс качества и времени обучения.
- **Главное про transfer learning:** Предобученные веса значительно ускоряют сходимость и повышают качество, особенно на небольших датасетах.
- **Главное про segmentation и метрики:** IoU — интерпретируемая метрика для сегментации; pretrained модели работают "из коробки", но дообучение может улучшить результат.
