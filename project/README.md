# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект демонстрирует применение методов и инструментов инженерии ИИ: работу с данными, сравнительный эксперимент ML-моделей, упаковку модели в контейнеризованный сервис, логирование и базовое тестирование.

---

## 1. Паспорт проекта

- **Название проекта:** Сервис предсказания энергоэффективности зданий (Energy Efficiency Prediction)
- **Авторы:** Астахова Дарья Сергеевна, Дьяков Андрей Геннадьевич
- **Группа:** ИВБО-20-24
- **Контакты:** Дарья — @Ladnopona | Андрей — @rizor666

- **Краткое описание:**  
  Проект решает задачу многовыходной регрессии для прогнозирования тепловой нагрузки здания (отопление Y1 и охлаждение Y2) на этапе проектирования. Используются открытые данные UCI Energy Efficiency, сравнительный анализ классических моделей и ансамблевого подхода (Stacking). Результат — контейнеризованный REST API на FastAPI с Pydantic-валидацией, бизнес-логикой оценки рисков и генерацией инженерных рекомендаций.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` и `pyproject.toml` / `uv.lock` – зависимости проекта.
- `report.md` – подробный отчёт (постановка, данные, эксперименты, выбор модели).
- `self-checklist.md` – чеклист самопроверки перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `exp01_eda.ipynb` – разведочный анализ и визуализации.
  - `exp02_model_selection.ipynb` – обучение, кросс-валидация, сравнение метрик.
- `src/` – основной модульный код:
  - `data/`, `features/`, `models/` – пайплайн подготовки и инференса.
  - `service/` – FastAPI приложение с эндпоинтами `/health`, `/predict`, `/info`.
  - `utils/`, `config.py` – утилиты логирования и загрузки YAML-конфигов.
- `data/` – открытый датасет `ENB2012_data.csv` (768 записей).
- `configs/` – конфигурации: `data.yaml`, `experiment.yaml`, `serving.yaml`, `.env.example`.
- `tests/` – pytest-тесты (API, данные, модели, бизнес-логика).
- `artifacts/` – сохранённые модели, метрики, графики и логи.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.11` (проект успешно работает на 3.14).
- Менеджер пакетов `uv` (рекомендуется) или `pip`.
- Docker (опционально, для контейнерного запуска).

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Через uv (рекомендуется, учитывает uv.lock)
uv sync

# Или через pip
pip install -r requirements.txt
```

---

## 4. Как запустить проект

### 4.1. Запуск обучения модели

Обучение и сравнение моделей реализовано в ноутбуках:
```bash
jupyter notebook notebooks/exp02_model_selection.ipynb
```
Или базовый пайплайн:
```bash
python -m src.train
```

### 4.2. Запуск сервиса (API)

**Вариант A: Локальный запуск**
```bash
python -m src.service --host 127.0.0.1 --port 8000
```

**Вариант B: Docker**
```bash
docker build -t energy-predictor:latest .
docker run -p 8000:8000 --name energy-api energy-predictor:latest
```

**Эндпоинты:**
- `GET /health` – проверка работоспособности.
- `POST /predict` – получение прогноза нагрузок и рекомендаций.
- `GET /info` – информация о загруженной модели.

**Пример запроса:**
```bash
curl -X POST http://localhost:8000/predict -H "Content-Type: application/json" -d "{\"X1\":0.98,\"X2\":514.5,\"X3\":294.0,\"X4\":110.25,\"X5\":7.0,\"X6\":2,\"X7\":0.0,\"X8\":0}"
```

- **Источник:** UCI Energy Efficiency Dataset (ENB2012).
- **Формат:** CSV, разделитель `;`, десятичный разделитель `,`.
- **Объём:** 768 записей, 8 признаков, 2 целевые переменные.

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Датасет небольшой (768 записей), что требует осторожности при экстраполяции на новые типы зданий.
- Пороги риска (`risk_thresholds`) заданы эвристически в конфиге, требуют калибровки на реальных инженерных данных.
- Отсутствует мониторинг дрейфа данных и версионирование моделей в продакшне.

**Планы по развитию:**
- Интеграция MLflow для трекинга экспериментов и хранения артефактов.
- Добавление Prometheus-метрик (latency, error_rate, prediction_count).
- Реализация эндпоинта `/retrain` и пайплайн переобучения по расписанию.
- Расширение датасета синтетическими данными для улучшения обобщающей способности.
