# Ноутбуки проекта

В этой папке хранятся Jupyter ноутбуки с экспериментами для проекта Energy Efficiency Prediction.

## Ноутбуки

### exp01_eda.ipynb
Разведочный анализ данных (Exploratory Data Analysis)

**Цель:**
- Первичный анализ датасета ENB2012_data.csv
- Изучение распределений признаков и целевых переменных
- Анализ корреляций между переменными
- Визуализация основных закономерностей

**Что сделано:**
- Загрузка данных с правильными разделителями (; и ,)
- Статистическое описание датасета
- Построение матрицы корреляций
- Визуализация распределений Y1 и Y2
- Анализ влияния категориальных признаков (X6, X8)
- Сохранение графиков в artifacts/figures/eda/

**Основные выводы:**
- Сильная корреляция между площадью и целевыми переменными
- Целевые переменные распределены близко к нормальному
- Категориальные признаки значимы для предсказания

### exp02_model_selection.ipynb
Сравнение и выбор моделей

**Цель:**
- Обучение и сравнение нескольких моделей регрессии
- Выбор лучшей модели для продакшн-сервиса

**Что сделано:**
- Разделение на train/test (80/20, random_state=42)
- Предобработка: StandardScaler + OneHotEncoder
- Обучение 4 моделей:
  - LinearRegression (baseline)
  - RandomForest
  - CatBoost
  - StackingRegressor (ансамбль)
- Кросс-валидация (5-fold)
- Расчёт метрик: MAE, RMSE, R²
- Сравнение результатов и выбор финальной модели
- Сохранение моделей в artifacts/models/
- Сохранение метрик в artifacts/metrics/

**Результат:**
Выбрана модель StackingRegressor с лучшими метриками:
- R² (Y1) = 0.9907
- R² (Y2) = 0.9990

notebooks/

## Как использовать

1. Установите зависимости:
   pip install -r requirements.txt

2. Запустите ноутбуки по порядку:
   - Сначала exp01_eda.ipynb (анализ данных)
   - Затем exp02_model_selection.ipynb (обучение моделей)

3. Результаты сохраняются автоматически в artifacts/

## Важные замечания

- Ноутбуки используют абсолютные пути через pathlib
- Конфигурации генерируются автоматически в configs/
- Для воспроизведения результатов используйте random_state=42
- Весь "боевой" код вынесен в модули src/
